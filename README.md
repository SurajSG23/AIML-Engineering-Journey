# AI/ML Learning Roadmap

This repository is a minimal, markdown-first workspace to track and practice AI/ML topics across three learning phases. It's intentionally lightweight so you can focus on notes, code experiments, and incremental progress.

## Purpose

- Provide a consistent folder layout for staged learning (foundations → GenAI & MLOps → production).
- Keep each topic focused with a single `practice.md` for notes, exercises, links, and short code snippets.

## Quickstart

1. Open a topic folder (for example: `phase-1-foundations/01-python-fundamentals`).
2. Edit `practice.md` to add examples, commands, short code snippets, and task checklists.
3. Update `roadmap-progress.md` to mark topics as you complete them.

## Repository structure

- `README.md` — this file (how to use the repo).
- `roadmap-progress.md` — checklist for every topic across phases.
- `phase-1-foundations/`, `phase-2-genai-and-mlops/`, `phase-3-production-ai/` — main phase folders.
- Each topic folder inside a phase is prefixed with a two-digit index and kebab-case name, for example: `01-python-fundamentals`.
- Each topic folder contains only `practice.md` (keep experiments and larger projects elsewhere if needed).

## Naming conventions

- Use kebab-case for folders and files (e.g., `data-versioning`).
- Topic folders are numbered within a phase using `01-`, `02-`, ... to preserve order.
- Keep filenames minimal: only `practice.md` inside topic folders.

## How to use `practice.md`

- Add a short header, goals, and 3–5 concrete tasks or exercises at the top.
- Keep runnable snippets small; include a command example for reproducing results.
- Use fenced code blocks for short scripts and commands.

Example snippet:

```bash
python -m venv .venv
source .venv/Scripts/activate
pip install -r requirements.txt
```

## Roadmap progress

- Use `roadmap-progress.md` to track completion: check the box for a topic when you've finished core exercises or a short project.
- Prefer granular progress updates (e.g., partially complete sub-tasks inside a topic's `practice.md`).
